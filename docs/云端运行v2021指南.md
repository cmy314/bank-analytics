# 本地内存不足 — 在 Kaggle / Colab / Drive 上跑 v2021

MSRTQps 与 MSResource 解压后 CSV 很大；**pivot + merge** 峰值内存常需 **8～16GB+**。  
本仓库已支持 **`MSRT_NROWS`、`MS_RESOURCE_NROWS`、`ONLY_FIRST_MSNAME=true`（默认）** 降规模。

---

## 一、先弄清内存花在哪

| 步骤 | 吃内存原因 |
|------|------------|
| `read_csv(MSRT)` | 行数 × 列数；全表可达数 GB |
| `pivot_table` | 长表变宽表，键 `(timestamp, msname, msinstanceid)` × 多种 metric |
| `read_csv(MSResource)` | 资源表也很大（已支持 `MS_RESOURCE_NROWS` + 按 msname 过滤） |
| `merge` | 宽表 × 资源表 |

**论文/开题实验**通常只需 **一个 msname + 一个分片（_0）**，不必全量 12 小时全服务。

---

## 四、Kaggle Notebook（推荐：免费 16GB RAM）

> **本机磁盘不够、要多分片拉长时间序列**：请直接看 **[Kaggle多分片运行指南.md](Kaggle多分片运行指南.md)**（OSS `wget` + `v2021-shards`，无需上传全量 CSV）。

### 4.1 创建私有 Dataset

1. 登录 [Kaggle](https://www.kaggle.com) → **Datasets** → **New Dataset**。  
2. 上传 `MSRTQps_0.csv`、`MSResource_0.csv`（可只上传 _0 分片，不必全部 MSRT/MSResource 包）。  
3. 记下数据集 slug，例如：`yourname/alibaba-v2021-shard0`。

> 两个 CSV 合计仍可能数 GB；上传较慢，但只需一次。Kaggle Dataset 单文件上限约 **20GB**（以平台当前规则为准）。

### 4.2 新建 Notebook

1. **Code** → **New Notebook**。  
2. 右侧 **Add Data** → 选你的 Dataset → 挂载到 `/kaggle/input/alibaba-v2021-shard0/`。  
3. **Settings** → **Accelerator: None**（不必 GPU）→ **Internet: On**（若要 `git clone`）。

### 4.3 Notebook 代码

## 六、参数建议（按 Kaggle 16GB RAM）

| 变量 | 建议起点 | OOM 时 |
|------|----------|--------|
| `MSRT_NROWS` | 1_000_000 | 降到 500_000 |
| `MS_RESOURCE_NROWS` | 3_000_000 | 2_000_000 |
| `ONLY_FIRST_MSNAME` | true | 保持 true |

`ONLY_FIRST_MSNAME=true` 时，**MSResource 会自动只保留同一 msname**，显著减小 merge 压力。

## 八、常见问题

**Q：Kaggle 上没有现成 Dataset？**  
A：自建 Private Dataset 最稳；公开集搜索 `alibaba microservices trace 2021`，有则可直接 Add Data，路径改为 `/kaggle/input/.../MSRTQps_0.csv`。

**Q：pivot 很慢？**  
A：正常；先减小 `MSRT_NROWS`，确认 `ONLY_FIRST_MSNAME=true`。

**Q：merge 后行数为 0？**  
A：MSRT 与 MSResource 分片号不一致，或 msname 过滤后无交集；换同一 `_0` 分片并重试。

**Q：Google Drive 断线？**  
A：大文件用 Colab 高 RAM + 输出写到 `/content/drive/.../output`，跑完再下载。
