# 将 analytics 拆成独立 Git 仓库

`analytics/` 原为 **bank-observability-demo** 下的 Python 子工程，与 Java 微服务、监控配置分离后便于：

- 单独推送到 GitHub / Gitee，Kaggle 只上传 `analytics.zip`
- 独立版本号、`pip install -e .`、私有 `.env`
- 论文附录只引用分析仓库，不拖整个银行 Demo

---

## 方案 A：复制为新仓库（推荐，最简单）

在 **bank-observability-demo 根目录** 执行：

```powershell
.\scripts\init-standalone-analytics.ps1
# 或指定目录：
.\scripts\init-standalone-analytics.ps1 -Dest F:\bank-analytics
```

会在同级生成 **`bank-analytics/`**（默认 `../bank-analytics`），已 `git init` 并完成首次提交（不含 `.venv`、`.env`、`output/`、`data/`）。

然后：

```powershell
cd F:\bank-analytics
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
# 编辑 .env 后
python -m bank_analytics v2021-shards --msrt-shards 0,1,2,3 --resource-shards 0,1
```

关联远程：

```powershell
git remote add origin https://github.com/你的用户名/bank-analytics.git
git branch -M main
git push -u origin main
```

## 方案 B：保留 Git 历史（仅 analytics 目录的提交）

需安装 [git-filter-repo](https://github.com/newren/git-filter-repo)：

```powershell
cd F:\bank-observability-demo
git clone . F:\bank-analytics-history
cd F:\bank-analytics-history
git filter-repo --subdirectory-filter analytics --force
```

得到的历史提交路径前缀为 `analytics/` → 拆出后文件在仓库根目录。

## 日常同步（主仓改 analytics → 独立仓）

主仓路径：`bank-observability-demo/analytics/`  
独立仓路径：`F:\bank-analytics\`（仓库根即原 analytics 内容）

在 **bank-observability-demo 根目录** 执行：

```powershell
.\scripts\sync-analytics-to-bank-analytics.ps1
```

会复制代码与文档，**不覆盖** 独立仓的 `.env`、`output/`、`data/`、`.venv/`、`.git/`。

同步后在独立仓提交并推送：

```powershell
cd F:\bank-analytics
git add -A
git commit -m "sync: analytics from bank-observability-demo"
git push
```

Kaggle 下次 `git pull` 或重新 `git clone` 即可用新代码。

## 主仓库（Java 项目）如何处理

三选一：

| 方式 | 做法 | 适用 |
|------|------|------|
| **1. 文档链接** | 删除或保留 `analytics/`，在 `docs/README.md` 写新仓库 URL | 彻底分离 |
| **2. Git Submodule** | 删除目录后 `git submodule add <url> analytics` | 仍要在一棵树下开发 |
| **3. 暂留双份** | 主仓保留只读副本，开发只在新仓 | 过渡期 |

### Submodule 示例（主仓库根目录）

```powershell
# 已把 analytics 推到远程后
git rm -r analytics
git commit -m "chore: move analytics to standalone repo"
git submodule add https://github.com/你的用户名/bank-analytics.git analytics
git commit -m "chore: add analytics submodule"
```

克隆主仓库的人需：

```bash
git clone --recurse-submodules <主仓库>
# 或
git submodule update --init --recursive
```

## 独立仓库目录结构（根即原 analytics）

```
bank-analytics/          ← 新仓库根（不是再套一层 analytics/）
  bank_analytics/
  docs/
  requirements.txt
  pyproject.toml
  README.md
  .env.example
  run_v2021.py
  run_v2021_shards.py
```

Kaggle / PyCharm 的 **Working directory** 设为该仓库根目录。

---

## 文档与链接需改的地方（主仓库）

拆出后请把主仓库里「`analytics/...`」的说明改为新仓库链接，例如：

- `docs/README.md`
- `docs/机器学习部分设计说明与实现说明.md`
- `docs/总体设计说明书.md`
- `ml/README.md`

独立仓内文档路径不变（仍为 `docs/...`）。

---

## 不要提交的文件

见仓库根 `.gitignore`：`.env`、`output/`、`data/`、`.venv/`。

`.env` 含本机路径，仅保留 `.env.example` 在 Git 中。
